* Let、const的暂时性死区

  使用let声明的变量只在let所在的的代码块内有效，且不存在变量提升，如果先使用再用let声明变量会报错。

  ​

  当一个块级作用域中存在let,const时，它便会绑定这一区域，在这一区域内，在声明前使用变量就会报错，这就是暂时性死区。

  ```
  if (true) {
    // TDZ开始
    tmp = 'abc'; // ReferenceError
    console.log(tmp); // ReferenceError

    let tmp; // TDZ结束
    console.log(tmp); // undefined

    tmp = 123;
    console.log(tmp); // 123
  }
  ```
